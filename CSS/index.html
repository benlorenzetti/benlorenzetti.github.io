Article

The webpage is designed with 6 major divisions:

  1. A <header> division, which is not used but could potentially hold a banner image.
  2. The <aside> semantic division is for the website's logo/brand image. It is floated left so it appears left or above the main content depeding on screen size.
  3. A <div> wraps around the article and navigation divisions. It holds the navigation bar horizontally aligned with the main page content. This container is floated left, following the aside image. The width is fixed to allow ~70 characters per line.
  4. The <article> contains the main content. The article's CSS can be customized for each page. This template has no CSS applied to the article.
  5. The <nav> division has a gradiated background image and uses floated, block style links to form a navigation bar. It contains an empty clear division to ensure it fills to the full width of its wrapper.
  6. The <footer> division should usually be left empty. It is necessary to clear the floats and extend the page height below the content. The background image varies vertically so that the page appears darker as one scrolls down.

There are essentially 5 divisions for content and 1 "wrapper" divisio|n for page layout. Only two wrapper division are assigned a fixed pix|el width. The article+nav wrapper should be assigned a width that lim|its paragraphs to 70 characters per line, for readability. Font-size |for the webpage should be assigned in pixels or point so that zooming| does not affect the word wrapping.

Hyperlinks should use relative URIs so the domain name can be changed without editing every page and to allow testing on local filesystems. Another reason absolute URIs should not be used is it would break my future localhost browser authentication scheme.
